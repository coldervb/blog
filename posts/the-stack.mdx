This blog is deliberately simple. Here's exactly what it uses and why.

## Next.js App Router

The App Router with `generateStaticParams` means every post is a static HTML file at build time. No server required, instant page loads.

## MDX files as posts

Each post is a `.mdx` file in `/posts` with frontmatter:

```mdx
---
title: The Stack
date: "2026-05-10"
description: Short description for the homepage listing.
tags: [meta]
---

Your post content here...
```

`gray-matter` parses the frontmatter, `next-mdx-remote` compiles the MDX into React components server-side.

## Tailwind + Typography plugin

The `@tailwindcss/typography` plugin handles post body styling. All prose styles are defined in `tailwind.config.ts`, targeting the dark background.

## Deployment

Push to GitHub, connect to Vercel — done. Vercel auto-detects Next.js and sets everything up. No `vercel.json` needed.

## Adding a post

1. Create `posts/my-post.mdx`
2. Add frontmatter
3. Write in Markdown / JSX
4. `git push`

That's it.
